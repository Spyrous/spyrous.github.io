The Famous Code Formatting Techniques

Aug 31st, 2014 7:12 pm

This topic bothered me a lot at the beginning of my programming journey. I wanted to find ‘the‘ best code formatting technique and stick with it so my code would be easy to read at any time. After a year or so it is obvious for me that there is no technique that you can say it is the best. It is more of a personal taste.

So should we invest time to find an acceptable style ? The answer is yes.

Fortunately there are a few ‘official’ conventions that you can pick from and get by. You can customize them a little if you think that something doesn’t look pretty but I would suggest you to endorse them as most of the programmers will be more used to reading these techniques and be able to read your code and help you more effectively.

Nevertheless your coding style comes second when you are involved in a programming project. You have to adapt to the style that everyone is writing. Although if you happen to start one on your own choose which technique will serve you and (if you happen to have) your colleagues best and keep it throughout it.

Here I introduce you a couple of sites that cover programming languages formatting techniques that are widely used today:

-> http://net.tutsplus.com/tutorials/html-css-techniques/top-15-best-practices-for-writing-super-readable-code/

-> http://www.gnu.org/prep/standards/html_node/Formatting.html

-> http://en.wikipedia.org/wiki/Programming_style

I would also encourage you to read through professional open source code like let’s say if you are a java programmer from guava. Guava is a library made by google’s professional software developers and it can give you a lot of hints and information about what is a good programming format and structure. There are endless sources of well-written code around the web. Examine it and build your programming style around solid foundations before you start coding.

Recapping ! Each Developer MUST follow the next principle:
Always code as if the person who ends up maintaining your code is a violent psychopath who knows where you live.

Don’t reinvent the wheel there are a couple of standards out there for many years which are held for a good reason.
